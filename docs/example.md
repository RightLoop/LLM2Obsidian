# 输入示例：

## 1. 长度获取错误

### 你出现的问题

一开始写了：

```
int len1 = sizeof(word1);
int len2 = sizeof(word2);
```

### 错误本质

你把 `char *` 当成了字符数组本体，误以为 `sizeof` 可以得到字符串长度。

### 反映出的薄弱点

这是 **指针与数组概念混淆**。

你还没有完全建立起下面这个区分：

- `sizeof(数组)` 得到的是整个数组占用字节数
- `sizeof(指针)` 得到的是指针变量本身大小
- `strlen()` 才是求字符串实际长度

### 应补的知识点

- 数组名与指针的关系
- 函数参数中的 `char word1[]` 和 `char *word1` 本质
- `sizeof` 与 `strlen` 的区别

## 节点2：

**链接关系：指向节点1**

**标题：**数组名与指针的关系

**内容：**

### 数组名**是**：

1. **数组的标识符**（编译器知道它是整个数组）
2. **常量表达式**（值在编译时确定，不能被修改）
3. **在大多数表达式中会"退化"为指向首元素的指针**

### 数组名**不是**：

1. **不是指针变量**（没有自己的存储空间来存放地址）
2. **不能进行赋值操作**（`arr = ...` 是非法的）
3. **不是左值**（不能出现在赋值运算符的左边）

## 📊 关键区别对比表

| 特性         | **数组名** (如 `int arr[5]`)          | **指针变量** (如 `int *ptr`)          |
| :----------- | :------------------------------------ | :------------------------------------ |
| **本质**     | 数组对象的标识符（符号表中的标签）    | 独立的变量，存储一个地址值            |
| **存储**     | **不单独占用存储空间**存储地址        | **占用存储空间**（4或8字节）存储地址  |
| **可修改性** | **常量**，不能重新赋值（`arr = x` ❌） | **变量**，可以重新赋值（`ptr = x` ✅） |
| **`sizeof`** | 返回**整个数组的字节数**              | 返回**指针本身的大小**                |
| **`&`操作**  | `&arr` 得到**数组指针** (`int(*)[5]`) | `&ptr` 得到**指针的指针** (`int**`)   |
| **算术运算** | `arr + 1` 按**数组元素大小**移动      | `ptr + 1` 按**指向类型大小**移动      |
| **函数传参** | **退化**为指针（丢失数组大小信息）    | 传递指针的副本（值传递）              |

## 底层视角：编译器如何看待数组名？

`int arr[5] = {1, 2, 3, 4, 5};`
`int *ptr = arr;`

`// 从编译器的视角看：`
`// arr: "我代表一个int[5]类型的数组对象"`
`// ptr: "我是一个变量，里面存了一个地址"`

`内存地址    内容         变量名      类型视角`
`0x1000  [ 1 ]       arr[0]      int`
`0x1004  [ 2 ]       arr[1]      int  
0x1008  [ 3 ]       arr[2]      int`
`0x100C  [ 4 ]       arr[3]      int`
`0x1010  [ 5 ]       arr[4]      int`
`0x1014  [0x1000]    ptr         int* (变量)`
`↑ ptr这个变量本身存储在0x1014，里面存的是0x1000这个值`
`↑ arr这个名字在编译时被翻译成地址0x1000，它没有自己的存储空间`

## 常见陷阱与理解要点

### 陷阱1：误以为数组名是指针变量

```
<C>
int arr[5];

int *ptr = arr;



// arr 没有自己的存储空间来存地址

// ptr 有存储空间（栈上4/8字节），里面存了arr的地址
```

### 陷阱2：对数组名进行赋值

```
<C>
int arr[5];

int *ptr;



ptr = arr;      // ✅ 合法：指针可以指向数组

// arr = ptr;   // ❌ 非法：数组名是常量，不能重新赋值
```

### 陷阱3：函数参数中的"退化"

```
<C>
void func(int arr_param[10]) {

    // 编译器看到的是：void func(int *arr_param)

    // 所以 sizeof(arr_param) == sizeof(int*)，不是40！

    // 这是无数新手困惑的地方

}
```

### 陷阱4：混淆`&arr`和`arr`

```
<C>
int arr[5];

// arr的类型是 int[5]，但在表达式中退化为 int*

// &arr的类型是 int(*)[5] （数组指针）

// 它们的值相同，但类型不同！
```

### 陷阱5：多维数组的"退化"

```
<C>
int matrix[3][4];

// matrix 退化为 int(*)[4]，不是 int**

// 这是理解多维数组传参的关键！
```

### 陷阱6：字符串字面量的特殊性质

```
<C>

char *str = "Hello";  // str是指向只读内存的指针

char arr[] = "Hello"; // arr是栈上的可修改数组



// str[0] = 'h'; // ❌ 运行时错误（字符串字面量只读）

// arr[0] = 'h'; // ✅ 合法
```
